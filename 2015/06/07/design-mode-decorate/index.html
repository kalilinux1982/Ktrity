设计模式 装饰模式

  • 装饰模式
  • 设计模式

可以在不改变一个对象本身的基础上给对象增加额外的新行为。

动态给一个对象增加一些额外的职责。比生成子类进行扩展更灵活

代码分析

半透明模式和透明模式的主要区别为：
半透明-> 无法调用新增的方法。->不能实现多重装饰。
但设计相对简单，而且在实际中，大多数装饰模式都是半透明的。

模式应用环境

  1. 可动态给一个对象增加功能，也可以动态地撤销
  2. 当不能采用继承的方式对系统进行扩充或采用不利于系统扩展和维护时。
    2.1 系统中大龄独立的扩展，为支持每一种组合将产生大量的子类，子类数呈现爆炸性增长
    2.2 类定义不能继承(如：final类)

模式应用

  1. javax.swing包中的JList构件不支持直接滚动
  2. java IO的InputStream和OutputStream
