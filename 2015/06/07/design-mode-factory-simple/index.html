设计模式 简单工厂模式/静态工厂方法模式

  • 简单工厂模式
  • 设计模式
  • 静态工厂方法模式

适用环境

  1. 创建的对象比较少，业务逻辑不太复杂
  2. 只知道工厂类的参数，对如何创建对象并不关心。 同一个类型的不同方式的处理

如：

  1. 创建不同形状的按钮，只需知道按钮类的一个参数，并提供一个调用方便的方法，传参数即可返回一个相应的对象
  2. 多种支付方式（现金，信用卡，代金劵），一个抽象支付类，每种支付一个类，进行相应的处理逻辑。在利用不同的参数创建不同的对象
  3. 权限管理，依靠不同的参数，进行分配不同的权限，有：管理员，经理，用户等进行区分。
  4. 多种方式的登陆，有：第三方的登陆，邮箱登陆，手机验证码的登陆

案例

用户权限管理，依靠不同的权限去执行不同的事务

从上面可以知道，每个不同权限的用户是一个类，每个类可以执行属于自己相关的工作。还有一种案例是《大话设计模式》里提到的计算器的用法，每个加法，减法，乘法，除法的操作都可以是一个类，然后使用静态方法，进行处理。

优点与缺点

缺点

  1. 增加类的个数
  2. 系统扩展困难
  3. 静态工厂方法，造成工厂角色无法形成基于继承的等级结构。
  4. 新增加产品，要对工厂类进行修改

java模式当中的应用

  1. java.text.DateFormat,处理英语或非英语的日期及时间格式
  2. java的加密技术，对称加密技术（KeyGenerator）和非对称加密技术（KeyPairGenerator），通过getInstance()静态工厂方法，根据不同参数传入，得到不同的秘钥生成器。
