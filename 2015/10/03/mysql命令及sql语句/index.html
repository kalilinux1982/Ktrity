如何指定数据库创建表

show databases;
use test; #使用show中显示的棋种一个数据库

如何导出数据库：（mysqldump）

找到你的mysql的位置，打开此路径的黑框控制台，输入：

mysqldump -u 用户名 -p 数据库名 > 存放位置

这里是-u并不是-r等。

扩展：

  • 导出远程数据库：

    mysqldump -h 192.168.11.210 -u root -p test>c:/test.sql

  • 导出一个表的结构，并且带表中的数据

    mysqldump -u 用户名 -p 数据库名 表名 > 导出的文件名

  • 导入数据库

    mysql server 5.1/bin > mysql -u root -p
    mysql server 5.1/bin > mysql>use ‘数据库’

然后使用source命令,后面参数为脚本文件(如这里用到的.sql):

mysql server 5.1/bin > mysql>source d:wcnc_db.sql

如何删除表(drop、delete、delete)

  • 使用drop

    drop table 表名
    此方式将删除表的结构被依赖的约束(constrain)、触发器(trigger)、索引(index)，为（ddl）不能回滚立即生效.

  • 使用delete
    为数据库操作语言(dml)，这个操作会放到 rollback segement 中，事务提交之后才生效。
    每次删除一行，并在事务日志中为所删除的每行记录一项

    delete from 表名 # 不带where是将删除所有数据

  • 使用delete
    同属（ddl）不能回滚，立即生效。
    删除表的所有数据，但保留表结构。
    只在事务日志中记录页的释放

    truncate table 表名
    相当于delete不带where的时候，但是比delete快。

  • truncate与delete比较：

  1. 在删除时如果遇到任何一行违反约束（主要是外键约束），TRUNCATE TABLE仍然删除，只是表的结构及其列、约束、索引等保持不变，但DELETE是直接返回错误
  2. 被外键约束的表，不能使用TRUNCATE TABLE
  3. 如果想保留标识计数值，请改用DELETE
  4. TRUNCATE TABLE不能用于参与了索引视图的表

参考：SQL truncate 、delete与drop区别

性能优化

  • SQL Server系列之 删除大量数据文中进行测试插入和删除的效率。效率最高的为：批量插入，使用TRUNCATE进行删除。应用到场景为：

    1. 保留需要的数据之新表中->TRUNCATE原表数据->还原之前保留的数据之原表中

    2. 新建表结构->批量插入需要保留的数据->DROP原表->重命名新表为原表

注意：比第一种方法略快点，因为省略了数据还原（即最后一步的数据恢复），但是稍微麻烦点，因为你需要创建一张和以前原有一模一样的表结构，包括基本列、属性、约束、索性等等。

  • between and用法

    0~200的范围:
    SELECT * FROM titles WHERE ytd_sales BETWEEN 0 AND 200
    或者：
    WHERE ytd_sales > 0 AND ytd_sales < 200

  • in 的用法（是否在in包含里面）

    SELECT * FROM Store_Information WHERE Store_Name IN (‘Los Angeles’, ‘San Diego’);

  • group by all用法

    条件：只有在 SELECT 语句还包括 WHERE 子句时
    在添加ALL 关键字，那么查询结果将包括由 GROUP BY 子句产生的所有组，即使某些组没有符合搜索条件的行。
    例子：
    查询结果：
    1班 72
    2班 75
    3班 NULL
    正确为：SELECT AVG(score) FROM test WHERE class<3 GROUP BY ALL class
    解释：因为出现了3班 NULL
    条件限制了选择的class<3 所以只有1 ，2班有成绩。但添加了all就会包含group by的所有行。

  • grant权限控制

    授权格式： grant 权限 to 角色
    使用：grant select,insert on userdb.userinfo to'zhangsan'@'localhost'
