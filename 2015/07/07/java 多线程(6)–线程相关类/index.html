java 多线程(6)--线程相关类

  • ThreadLocal
  • java
  • 多线程

ThreadLocal

其功能非常简单：为每一个使用该变量的线程都提供一个变量值的副本，每个线程都可以独立地改变自己的副本，而不会和其他线程产生冲突。
提供三个public方法

  • T get() 返回当前副本的值
  • remove 删除此线程局部变量中当前线程的值
  • set() 设置此线程局部变量中当前线程副本中的值

详细使用参考：深入研究java.lang.ThreadLocal类

包装线程不安全的集合

使用collections提供静态方法把线程不安全的集合包装成线程安全的集合。分别为synchronized+对应的集合名

线程安全的集合类

以Concurrent开头的集合，支持并发访问的集合，支持多个线程并发写入访问，写入是线程安全，但读取不必锁定，采用复杂的算法并不会锁定整个集合。

  • ConcurrentLinedQueue,使用在多个线程共享访问一个公共集合时。不许使用null值，访问无须等待
  • ConcurrentHashMap支持16个线程并发写入，超过16个，需要等待，可以设置concurrentcyLevel构造参数来是更多的并发写入线程（默认16个）

缺点：当使用迭代器遍历集合时，该迭代器可能不能反映创建迭代器之后所做的修改，但程序却不会抛出异常。

以CopyOnWrite开头的集合

  • CopyOnWriteArrayList 复制底层的数组的方式来实现写操作。执行读操作时不必加锁和阻塞。当执行写入操作的时候（add,set,remove）会对数组的副本进行操作，所以是线程安全的。当执行写入操作的时候需要频繁地复制数组，性能比较差，但读操作就很快，很安全。所读>写。适用在缓存
