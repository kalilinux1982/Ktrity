java-探讨System.out.println(数组);的结果

2015-07-28
java

直接使用System.out.println(数组);会有怎么样的结果呢？为什么字符数组是显示完全部元素呢？

测试：

唯有char[]数组打印为ab，其他打印乱七八糟的字符串呢？
查看API发现

void print（char[] x）
打印字符数组，然后终止该行。

查看api说了这样就这样有点不爽，我们看看源码吧。
输出字符数组的情况

最后是使用流的方式进行每一个字符输出。所以最后就显示字符数组里的所有元素了

而其他类型的数组的时候，查看源码就知道都归为Object,如下：

可以发现，最终是obj.toString()搞的鬼。

除了字符数组，其他类型的输出，最终都决定于obj.toString()

下面引用：java 不用遍历，直接打印出来char类型数组,这里的解释：

jdk中的解释是 println(char[] value) 直接饮用了print(char[] s)，而print(char[] s)解释为：打印字符数组。按照平台的默认字符编码将字符转换为字节，并完全以 write(int) 方法的方式写入这些字节。和打印String用了同样的方法。print(String value)的解释为：打印字符串。如果参数为 null，则打印字符串 “null”。否则，按照平台的默认字符编码将字符串的字符转换为字节，并完全以 write(int) 方法的方式写入这些字节.
所以打印字符数组和直接打印字符串是相同的。
