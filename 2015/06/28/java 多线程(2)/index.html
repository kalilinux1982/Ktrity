控制线程

join线程

Thread提供了让一个线程等待另一个线程完成的方法—join()方法。

某个执行流中调用其他线程的join()方法时，调用线程被阻塞，直到被join()方法加入的join线程执行完为止。

简单的例子理解：将大问题划分成许多小问题，每个小问题分配一个线程。当所有的小问题都得到处理后，再调用主线程进一步操作。

实验代码如下：

程序执行时，名为新线程会和主线程main进行并发执行。当创建被join的线程的时候，主线程main将被阻塞，指定被join的线程线程执行完毕，才会继续执行。在执行完毕前，新线程和被join的线程并发执行。

join有3中重载形式

  • join(); 得带被join的线程执行完成。
  • join(long millis); 等待被join的线程的时间最长为millis毫秒。如果在millis毫秒内被join的线程还没有执行结束，则不再等待。
  • join(long millis); 等待被join的线程的时间最长为millis毫秒+nanos毫微秒。不过，很好使用，计算机硬件，操作系统本身无法精确到毫微秒

应用：

主线程生成并起动了子线程，而子线程里要进行大量的耗时的运算(这里可以借鉴下线程的作用)，当主线程处理完其他的事务后，需要用到子线程的处理结果，这个时候就要用到join();方法了。

后台线程/守护线程/精灵线程

JVM的牢记回收线程就是典型的后台线程。

特征：如果所有的前台线程都死亡，后台线程会自动死亡。所以，会经常出现此情况：前台线程是保证执行完毕的，后台线程还没有执行完毕就退出了。

调用Thread对象的setDaemon(true)方法将制定线程设置成后台线程。

注意：

  • isDaemon()方法，判断制定线程是否为后台线程
  • 主线程默认为前台线程。
  • 前台线程创建的子线程默认是前台线程，后台线程创建的子线程默认为后台线程。
  • setDaemon()必须在start()方法之前调用

其中一种应用的方式就是：当输入输出阻塞的时候，后台线程还在继续执行。

线程睡眠：sleep()

让当前正在执行的线程暂停一段时间，并进入阻塞状态。在睡眠时间内，该线程不会获得执行的机会

线程让步：yield()

让当前正在执行的线程暂停，但它不会阻塞该线程，它只是将该线程转入就绪状态。

yield()只是让当前线程暂停一下，让系统的线程调度器重新调度一次。完全有可能的情况：使用yield()方法暂停后，线程调度器又将其调度出来重新执行。

使用yield()暂停后，只有优先级与当前线程相同或者优先级比当前高的处于就绪状态的线程才会获得执行的机会。

yield和sleep区别

  1. 使用sleep后，不理会其他线程的优先级，但使用yield，只有优先级相同或更高才有机会执行。
  2. sleep后，进入阻塞状态，经过阻塞时间才会转入就绪状态。而yield直接为就绪状态。
  3. sleep比yield更好的移植，通常不建议使用yield来控制并发线程的执行

改变线程优先级setPriority()

每个线程都具有一定的优先级，优先级高的获得较多的执行机会，而优先级低的线程则获得较少的执行机会。

每个线程默认的优先级都与创建它的父线程的优先级相同。默认，main具有普通优先级。

setPriority()的参数，范围1~10，有3个静态常量才适应全系统

  • MAX_PRIORITY 值10
  • MIN_PRIORITY 值1
  • NORM_PRIPRITY 值5
