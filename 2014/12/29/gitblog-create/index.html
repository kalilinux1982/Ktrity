Github博客的搭建

2014-12-29
hexo博客

为了装逼，就弄了个github博客来玩。

吐槽

好不容易，终于搭起了github博客，根据README.md一步步进行步骤的搭建

如何搭建博客
1.fork库到自己的github
2.修改名字为：username.github.io
3.clone库到本地，参考_posts中的目录结构自己创建适合自己的文章目录结构
4.修改CNAME，或者删掉这个文件，使用默认域名
5.修改_config.yml配置项
6.It's done!

特别注意到第2点修改名字为：username.github.io中的username必须跟你的gitub账号名一致，否则，将会创建不成功，不要问我为什么，我也不知道，今天边看魁拔妖侠传，边一次次的试。结果看完了全部，也就在最后一次中成功了。

感谢简书上的@张明云发给我的简信中，知道了这么直接的github博客

最简单的办法是，在Github上找到别人的博客作为模板，比如suyan，直接fork到你的名下，然后就按照它ReadMe.md里面介绍的步骤来改，这是最简单的。

感谢最原始的博主SuYan0830的热心回答

如何将购买的域名，指向此github博客，成为自己的个人博客。

  • 参考：
    My custom domain isn’t working
    Tips for configuring an A record with your DNS provider
    全是英文，误打误撞，结果行了，真心觉得英语不好，就悲剧，论如何提高英语？

  • 步骤1：

    If you are using a subdomain, like www.example.com or blog.mydomain.com, you should have a CNAME record pointing to username.github.io
    就是讲你的username.github.io进行域名解析

  • 步骤2：

    Configuring an A record with your DNS provider
    With your DNS provider, create A records that resolve to the following IP addresses:
    1：192.30.252.153
    2：192.30.252.154

如博客不能登录，有可能是github更改了空间服务的ip地址，记得及时到在GitHub Pages查看最新的ip即可.

添加评论，升级版

经过几番折腾，看了一遍又一遍的认为可能的代码~~page.html，结果还是没有任何效果，无助，只能求助作者，终于在评论中，找到了添加评论的方法，现在有点后悔当初学web的时候，为什么趁着去学js呢？有得必有失.出来混，迟早都是要还的

  • 参考siberiawolf.github.io和Lingxiangkong.github.io的添加评论的方式
  • 才知道是在post.html中添加相应disqus的代码，才发现post.html是指定内容页的全部布局和样式，详细查看post.html
  • 要想用到disqus的评论接口和插件，必须进行注册。
    • 点击disqus入口
    • 点击Get disqus for you site按钮，进行相应的信息的填写注册

      • 注册后会弹出多个博客平台，选择Universal Code即可
    • 接着，会教你如何进行部署，部署的步骤,只要你按着步骤来的话，即可直接copy代码部分，粘贴到post.html适合的位置中。否则在var disqus_shortname = 'example';的example填写你已经注册的site
