java-Map集合-HashMap

2015-07-28

Map集合就想是Set集合和List集合的组合。
为什么这样说呢？
因为：Map<K,V>中key是无序不可重复的集合相当于Set集合，其中Map的一个方法KeySet()就是返回所有key组成的set集合。
而value是有序，可重复的集合，相当于List集合。
为什么说是有序呢？这里的value对应的索引为key而不是List集合中单一的整数值。

HashMap和HashTable的区别

两者主要的区别在于：

  1. Hashtable是线程安全的，而HashMap是线程不安全的，当多线程访问同一个Map对象时，使用Hashtable实现类会更好。
  2. Hashtable不允许使用null作为key和value,但HashMap可以使用null作为key或者value

为什么一定要实现hashCode方法和equals方法？

让我们看看基类Object的这两个方法

我们可以知道只有当且仅当同一个对象所处的内存地址相同时的时候，才会认为相同。例如：
第一种情况：new Object()和new Object()为两个对象，放在HashMap中是可以存在的，那么就达不到我们预想的结果：key不可重复。
另一个种情况：从Object的hashCode方法可以得知，它是返回当前对象的内存地址，当我们使用put存进去的时候，内存地址变化了，产生不同于当初的键。而使用get的方式，加入参数进行匹配时候，当前的对象即键已经改变，导致无法找到。
所以我们基于第一种情况必须重写equals方法和第二种情况需要hashCode方法.

注意：在Map集合使用过程中：equals()相等，hashCode就一定要相等，而hashCode相等，却不要求equals相等，为什么呢？

Map的结构能够快速找到一个对象，而不是进行较慢的线性查找。使用hash过的键来定位对象分两步。Map可以看作是数组的数组。第一个数组的索引就是对键采用hashCode()计算出来的值，再在这个位置上查找第二个数组，使用键的equals()方法来进行线性查找，直到找到要找的对象。参考：Java中的equals()和hashCode()契约

HashMap的调用put函数操作

1.首先根据put元素的key获取hashcode,然后根据hashcode算出数组的下标位置,如果下标位置没有元素，直接放入元素即可。
2.如果该下标位置有元素（即根据put元素的key算出的hashcode一样即重复了），则需要已有元素和put元素的key对象比较equals方法，如果equals不一样，则说明可以放入进map中。这里由于hashcode一样，所以得出的数组下标位置相同。所以会在该数组位置创建一个链表，后put进入的元素到放链表头，原来的元素向后移动。

与HashMap存储数据时的比较：

IdentityHashMap和HashMap内部都实现了散列表，但有区别，体现在对散列冲突的处理上。HashMap中以分离链表的方式来解决散列冲突，也就是将散列在同一个桶内的数据组织成一个链表结构；IdentityHashMap中则以开放寻址的方式来解决散列冲突，当发生散列冲突时，数据被放入下一个空闲地址(也叫线程探测法)
