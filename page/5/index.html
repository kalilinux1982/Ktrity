      LayoutInflater.from(context).inflate(R.layout.title, this);

    • 源码解析其中的原理
  • layout_width和layout_height要在布局包裹下才起作用，默认为FrameLayout.

Android视图绘制流程完全解析，带你一步步深入了解View(二)
知识点：

  • onMeasure 测量视图大小
  • onLayout 对子视图进行布局（不同布局如relativeLayout的规则都不同）
  • onDraw 绘制view的流程

getMeasureWidth和getWidth的区别：

getMeasureWidth()方法:
在measure()过程结束后就可以获取到
通过setMeasuredDimension()方法来进行设置的.
getWidth()方法:
要在layout()过程结束后才能获取到
通过视图右边的坐标减去左边的坐标计算出来的

一般情况下，两者的数据相同。

Android视图状态及重绘流程分析，带你一步步深入了解View(三)
知识点：

  • 视图状态：（每个view都拥有）

    1. enabled 是否可用，响应onTouch事件
    2. focused 获取焦点，一般只有视图在focusable和focusable in touch mode同时成立，才可以获取焦点。如editText
    3. window_focused 是否处于正在交互的窗口,系统自行决定
    4. selected 是否处于选中状态
    5. pressed 是否处于按下状态

  • 视图重绘
    setVisibility()、setEnabled()、setSelected()等方法导致视图重绘，可以手动调用invalidate或者postinvalidate来进行视图重绘，注意只会重绘draw流程，而不会重绘meause和layout流程，因为并没有改变大小和布局。
    但在这篇Android View 中invalidate() 你所不知道的那些事文章提出了异议。

    invalidate()会触发view Layout(),和draw(),但是它不会进行测量。 requestLayout(）方法最终也会执行draw（）的。

Android自定义View的实现方法，带你一步步深入了解View(四)
知识点：

  • 自定义view：自动记录点击事件的btn
  • 组合控件：通过继承布局的view来加载布局以及相关的组件
  • 继承控件，通过继承view来处理相应的事件，如listview中添加滑动删除的按钮，并实现相关的事件

postInvalidate和invalidate的区别

  • invalidate智能在UI线程中直接使用，即：oncreate的时候，其他时候需要消息机制（使用Handler或者runOnUiThread）。

  • postInvalidate直接封装了handler的消息机制，直接调用即可。

两者：

  • 在非UI线程中是建立在Thread异步更新上。（都需要线程进行执行）
  • 在UI线程中直接执行即可更新。并不需要写线程，handler之类的。

扩展（requestLayout）

  • 当view确定自身已经不再适合现有的区域时，该view本身调用这个方法要求parent view重新调用他的onMeasure和onLayout来对重新设置自己位置。（特别的当view的layoutparameter发生改变，并且它的值还没能应用到view上）

用途：有时我们在改变一个view 的内容之后 可能会造成显示出现错误，比如写ListView的时候 重用convertview中的某个TextView 可能因为前后填入的text长度不同而造成显示出错，此时我们可以在改变内容之后调用requestLayout方法加以解决。

另外需要主要讲到的两中情况：

  • 注册了ontouch并返回true：

    每次点击它的时候都会触发一系列的ACTION_DOWN，ACTION_MOVE，ACTION_UP等事件。这里需要注意，如果你在执行ACTION_DOWN的时候返回了false，后面一系列其它的action就不会再得到执行了.

  • Android事件分发机制完全解析，带你从源码的角度彻底理解(下)

以上注意几个信息：
1. onTouch和onTouchEvent有什么区别，又该如何使用？

从源码中可以看出，这两个方法都是在View的dispatchTouchEvent中调用的，onTouch优先于onTouchEvent执行。如果在onTouch方法中通过返回true将事件消费掉，onTouchEvent将不会再执行。

我们需要关注下面的三个类：

  1. public boolean dispatchTouchEvent(MotionEvent ev) 这个方法用来分发TouchEvent
  2. public boolean onInterceptTouchEvent(MotionEvent ev) 这个方法用来拦截TouchEvent
  3. public boolean onTouchEvent(MotionEvent ev) 这个方法用来处理TouchEvent

对应的解释为：

当TouchEvent发生时，首先Activity将TouchEvent传递给最顶层的View， TouchEvent最先到达最顶层 view 的 dispatchTouchEvent ，然后由 dispatchTouchEvent 方法进行分发，如果dispatchTouchEvent返回true ，则交给这个view的onTouchEvent处理，如果dispatchTouchEvent返回 false ，则交给这个 view 的 interceptTouchEvent 方法来决定是否要拦截这个事件，如果 interceptTouchEvent 返回 true ，也就是拦截掉了，则交给它的 onTouchEvent 来处理，如果 interceptTouchEvent 返回 false ，那么就传递给子 view ，由子 view 的 dispatchTouchEvent 再来开始这个事件的分发。如果事件传递到某一层的子 view 的 onTouchEvent 上了，这个方法返回了 false ，那么这个事件会从这个 view 往上传递，都是 onTouchEvent 来接收。而如果传递到最上面的 onTouchEvent 也返回 false 的话，这个事件就会“消失”，而且接收不到下一次事件。

如何在java中配置BC库（BouncyCastle）

问题：

Java平台默认识别jks格式的证书文件，但是android平台只识别bks格式的证书文件，

所以需要自己配置BC库。在stackoverflow找到答案
How to create a BKS (BouncyCastle) format Java Keystore that contains a client certificate chain，在这里进行引用。（答案是android与tomcat的配置）

  1. 首先下载资源：bcprov-ext-jdk15on-1.46.jar

  2. 将bcprov-ext-jdk15on-1.46.jar复制到以下路径中。（每个对应的路径都不同）

    D:\tools\jdk1.5.0_09\jre\lib\ext (JDK (bundled JRE)
    D:\tools\jre1.5.0_09\lib\ext (JRE)
    C:\ (为了加入到环境变量中，可以改路径)

  3. 进行修改java.security路径如下：

    D:\tools\jdk1.5.0_09\jre\lib\security
    D:\tools\jre1.5.0_09\lib\security

在此文件中加入：security.provider.11=org.bouncycastle.jce.provider.BouncyCastleProvider，其中11是因人而异，按着文件中的顺序来的。

  1. 加入到环境变量中：CLASSPATH=%CLASSPATH%;c:\bcprov-ext-jdk15on-1.46.jar

  2. 将bcprov-ext-jdk15on-1.46.jar加入到你的工程中，并使用以下代码进行测试：

  3. 现在我们便可以使用keytool生成bks格式的了

    keytool -genkey -alias myproject -keystore C:/myproject.keystore -storepass myproject -storetype BKS -provider org.bouncycastle.jce.provider.BouncyCastleProvider

这时候会抛出以下错误（回答也遇到了）：java.security.InvalidKeyException: Illegal key size or default parameters。如何修改呢？
首先下载资源：Link to files for Java 7，以后进行替换。
在什么地方替换呢？
参考：AES加密时抛出java.security.InvalidKeyException: Illegal key size or default parameters

{java_home}/jre/lib/security/
替换相关的类

以上完成了BouncyCastle的配置和测试，但又会有一个问题，当你是使用android studio来编译客户端的时候，将会报这个错误：

从
Android studio App executing error
Problems launching error in Android Studio when packageDebug at Max OS X
的两个答案中告诉我们，之前我们在D:\tools\jdk1.5.0_09\jre\lib\ext加入bcprov-jdk15on-1.48.jar和android studio自带的jar产生冲突了，所以我们需要之前配置加入的bcprov-jdk15on-1.48.jar删去。

所以，最佳的方式为（不需要以上的配置步骤，当需要生成bks格式的使用便使用bcprov-jdk15on-1.48.jar即可）

  1. 在bcprov-jdk15on-1.48.jar同一个文件夹下
  2. 用 keytool 生成 keysotre 文件

    $ keytool -genkey -alias codeproject -keystore codeprojectssl.keystore -validity 365
    需要注意：当要求你录入 Common name 的时候, 要填你的主机名. 本文例子用的是: codeproject.com （在web对应配置下）

  3. 将密钥(key)从 .keystore 文件导入 .cer 文件

    $ keytool -export -alias codeproject -keystore codeprojectssl.keystore -file codeprojectsslcert.cer

  4. 最后是关键，生成.bks格式的文件

    $ keytool -import -alias codeproject -file codeprojectsslcert.cer -keystore codeprojectssl.bks -storetype BKS -providerClass org.bouncycastle.jce.provider.BouncyCastleProvider -providerpath bcprov-jdk15on-146.jar

使用-providerpath来执行bcprov-jdk15on-146.jar的路径。
