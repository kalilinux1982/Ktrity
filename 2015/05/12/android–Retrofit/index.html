参数 说明 备注
u_id 用户ID 数字。游客用0表示
action 获取方式 数字。1.获取最新的10条动态(无需指定动态id)；2.获取指定动态id早前的10条动态；3.获取比指定动态ID还要新的所有动态

怎么处理多个参数呢？个人觉得入门的一个框架的使用可以去看别人的博客，因为配置的问题，常常会出现一系列的问题，看到别人配置成功，去仿造一般不会有问题，但后面的开发，遇到相应的需求，通过网络进行百度，谷歌已经不能解决问题了，直接看api文档，官方的介绍来的更容易。通过查看官方的wiki了：A type-safe REST client for Android and Java，我们可以知道，对于多个参数，我们需要使用Map集合，进行处理：

For complex query parameter combinations a Map can be used.
@GET(“/group/{id}/users”)
List groupList(@Path(“id”) int groupId, @QueryMap Map options);

最后，参考代码

//获取动态
       Map<String,String> map=new HashMap<String, String>();
       map.put("u_id","0");
       map.put("action","1");

       RestAdapter restAdapter = new RestAdapter.Builder()
               .setEndpoint(Common.API_URL).setClient(new TimeOutUrilConClient()).setLogLevel(RestAdapter.LogLevel.FULL)
               .build();

       UriApi<String, Object> uriApi = restAdapter.create(UriApi.class);

       uriApi.getData(Common.TOPIC_PATH, map, new Callback<Object>() {
           @Override
           public void success(Object o, Response response) {
               Log.e("获取动态",o.toString());
               swipeRefreshLayout.setRefreshing(false);//完成后取消动画
           }

           @Override
           public void failure(RetrofitError error) {
               Log.e("获取动态失败",error.getMessage());
           }
       });

实战5：面对LinkedTreeMap无法转换成指定的类型

最近在使用第三方，微博登录后，试法获得当前用户的信息，结果返回的json，遇到了这个问题，我试调了一天，非常郁闷，我按着指定的json数据到在线转换进行转换指定的类，结果总是报错：总是说无法转换成指定的类型：cannot be cast to。数据为：

这是使用Retrofit框架会在logcat中自动输出的。既然这样，我就让他成为基类Object，然后再进行toString输出成字符串。再这个字符串使用gson进行转换成对应的类，结果：失败。
无奈，只能对toString的数据输出，结果发现，这个字符串不要用""存在，而上面的是用防转义的\"让他继续存在，使得转化成功的。突然觉得非常坑爹，居然无法转换的事情都该我遇上了。
最后，灵光一闪：.LinkedTreeMap到底是何方什么物，如果是属于Map集合，那么就有key,value键值对存在，而且上面一大串json数据，我并不是全部都要的，只是要一些基本信息而已。结果查看源码，真的还是我想象的一样，还真的成功了。
