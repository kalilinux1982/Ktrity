java 多线程(5)--线程池

  • java
  • 多线程
  • 线程池

系统 启动一个新线程的成本比较高，尤其当程序需要大量生存期很短的线程时，更需要考虑使用线程池。

使用线程池可以有效地控制系统并发线程的数量，当系统中包含大量的并发线程时，会导致线程性能剧烈下降，甚至导致JVM崩溃，而在线程池的最大线程数可以控制系统中并发线程数不超过次数。

java 5 实现的线程池

使用Executors工厂类来产生线程池

  • newCacheThreadPool 创建一个具有缓存功能的线程池，内部实现是SynchronousQueue 同步队列
  • newFixedThreadPool 创建一个可重用的，具有固定线程数的线程池，其内部实现为LinkedBlockingQueue
  • newSingleThreadExecutor 创建一个只有单线程的线程池，相当于调用newFixedThreadPool传入参数为1
  • newScheduledThreadPool 创建指定数量的线程数的线程池，指定延迟后执行线程任务，及时是空闲的线程也会保存在线程池中。

上面返回的是ExecutorService，
提供sumbit方法，方法返回future,参数可为callable/runnable，当使用runnable没有返回值，并不知道几时结束，可以使用isDone/isCancel来获得runnable的执行状态。

ScheduledExecutorService
指定延迟后或周期性地执行线程任务的线程池。方法为schedule进行延迟执行。

注意：用户线程池后，应该调用shutdown方法，关闭线程池，不再接收新任务，但会将以前所有已提交的任务执行完成。

java 7 新增ForkJoinPool

为了利用多个CPU的优势，把一个任务拆分成多个小任务进行并行计算，再把多个小任务的结果合并成总的计算结果。

ForkJoinPool是ExecutorPool的实现类

  • 构造函数：ForkJoinPool可以通过Runtime.availableProcessors的返回值，即：计算此电脑支持多个线程，例如我的是双核四线程的。
  • 使用submit和invoke来执行指定任务
  • 两个抽象子类：RecursiveAction(没有返回值的任务)和RecursiveTask(有返回值，支持泛型)
