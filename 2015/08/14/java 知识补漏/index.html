Stack栈和Heap堆的区别

堆：

Java 的堆是一个运行时数据区,类的(对象从中分配空间。这些对象通过new、newarray、anewarray和multianewarray等指令建立，它们不需要程序代码来显式的释放。堆是由垃圾回收来负责的，
堆的优势是可以动态地分配内存大小，生存期也不必事先告诉编译器，因为它是在运行时动态分配内存的，Java的垃圾收集器会自动收走这些不再使用的数据。
但缺点是，由于要在运行时动态分配内存，存取速度较慢。”

栈：

栈的优势是，存取速度比堆要快，仅次于寄存器，栈数据可以共享。
但缺点是，存在栈中的数据大小与生存期必须是确定的，缺乏灵活性。栈中主要存放一些基本类型的变量（,int, short, long, byte, float, double, boolean, char）和对象句柄。

所以，GC是针对堆Heap的

在多线程环境下，SimpleDateFormat 是线程安全的吗？(答案)

不是，非常不幸，DateFormat 的所有实现，包括 SimpleDateFormat 都不是线程安全的，因此你不应该在多线程序中使用，除非是在对外线程安全的环境中使用，如 将 SimpleDateFormat 限制在 ThreadLocal 中。如果你不这么做，在解析或者格式化日期的时候，可能会获取到一个不正确的结果。因此，从日期、时间处理的所有实践来说，我强力推荐 joda-time 库。

文件锁（FileLock）

fileChannel提供lock()/tryLock()方法获得文件所FileLock对象，从而锁定文件。

  • lock():试图锁定，无法锁定则阻塞。
  • tryLock():试图锁定，锁定返回文件锁，无法锁定返回null.可以设置参数shared,如果true为共享锁，如果false为排它锁。
  • release()释放文件锁。

注意在不同平台的使用注意问题：

  • 某些平台文件锁是建议，意味着，即使不能获得文件锁也可以进行读写。
  • 某些平台，不能同步地锁定一个文件并把他映射到内存中
  • 两个java程序使用同一个java虚拟机运行，则他们不能对同一个文件进行加锁。
  • 某些平台，关闭FileChannel，会释放java虚拟机上对该文件的所有锁，避免对同一个被锁定的文件打开多个FileChannel
