java 网络编程 TCP协议

2015-08-31
java

两端各建立一个socket，并形成虚拟链路。然后通过虚拟链路进行通信，java通过socket产生的IO流进行通信。

TCP协议基础

IP协议只保证计算机中能发送和接收数据。
负责将将数据分割成一个个小包，从一台主机传送到另一台。

TCP协议
端对端。收集信息包后，按适当的次序进行发送，接收端进行还原。
使用重发机制。发送后，需要收到确认的信息，否则进行重发。即使网络阻塞的情况也能得到可靠的通信。

原理解析：

三次握手

  • A->B 含有同步序列号的标志位 请求建立连接
  • B->A 确认应答(ACK)和同步序列号(SYN)标志位
  • A->B 再发送一个确认应答 可传输数据

使用ServerSocket 创建TCP服务器端

使用的类为ServerScoket，需要的参数为IP地址和端口，端口的设置范围：0~65535。（推荐1024以后端口，避免冲突）
循环使用Serversocket的accept方法，不断地监听和接收客户端发来的信息。改方法返回一个socket，接收了此socket便可以进行通信。accept方法将导致线程阻塞。
